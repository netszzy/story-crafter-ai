# V5.0-alpha1 附录对照复检

> 复检日期：2026-05-06
> 对照文档：`docs/V5.0_DESIGN_SOUL_APPENDIX.md`
> 复检范围：当前 V5.0-alpha1 改动，包括安全诊断、mock 警示、今天页、导航精简、inline 改稿预览、长任务提示和设置页连通测试。

## 1. 总结论

V5.0-alpha1 可以作为第一迭代继续推进，但对照附录只能判定为“部分达标”，不能宣称已完整实现 V5.0 体验灵魂。

alpha1 已经完成最关键的方向修正：score 开始带 caveat，氛围/内省章节的冲突类诊断可被降级，mock 不再假装能评文学性，打开 WebUI 默认进入“今天”，部分“采纳此改法”路径从正文目录辅助稿改为 inline 预览。

仍需继续补足的核心差距：真正后台任务尚未全面迁移，inline diff 仍是 Streamlit 原生预览而不是段级组件，卷轴健康图和 StyleProfile/LiteraryView/style_court 尚未落地，作家裁决权还没有进入所有后续改稿链路。

## 3. 关键风险

1. `score_caveat` 已经存在，但能否触发依赖任务卡里的 `chapter_mode`、`pacing`、`ending_style`、`style_profile`。alpha2 必须把这些字段变成稳定 schema 和 UI 入口，否则 B.1 的保护只会停在代码里。
2. “采纳此改法”已减少正文目录辅助稿，但仍保留审核日志和旧式 AI 辅助草案入口。后续需要明确区分“写作中即时改稿”和“可归档的策划辅助包”。
3. 长任务目前更多是提示层修正，不是真正后台执行。附录把“不锁屏”视为体验底线，后续不能把这项标记为完成。
4. 现在的 diff 仍是技术型 unified diff。附录要求的是作家看得懂的段落级改法，beta1/rc1 必须引入更贴近稿纸的组件。
5. 全书健康图必须避免红黄绿审判感；后续实现前要先废弃“32x32 像素格”思路。

## 4. alpha1 补强记录

- 已将附录原样保存为 `docs/V5.0_DESIGN_SOUL_APPENDIX.md`。
- 已在 `docs/V5.0_UPGRADE_PLAN.md`、`AGENTS.md`、`CLAUDE.md` 标记附录为 V5.0+ 升级前必读。
- 已把“今天”页摘录长度调整到 200 字，并把继续写卡片圆角调整到附录指定的 12px。
- 已把供应商连通测试默认超时降为 3 秒，并在保存设置时并行测试三家供应商。

## 5. 下一版本前硬门槛

V5.0-alpha2 开始前必须先读附录，并至少完成以下硬门槛：

1. ChapterMode/StyleProfile 成为稳定 task card schema，不再只靠临时字段。
2. 诊断 finding 需要带作家可裁决状态：采纳、保护、反驳。
3. 写作页即时改稿继续收敛到段级 inline diff，不新增正文目录辅助稿。
4. 长任务至少迁移一条真实高频路径到后台 job，而不是只显示非阻塞提示。
5. 新增以第001章或等价氛围章节为 fixture 的回归测试，证明系统不会再把氛围章节判成必须补冲突的失败章节。
