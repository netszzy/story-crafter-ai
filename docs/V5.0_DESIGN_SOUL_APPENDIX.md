# 附录 A · 设计灵魂卡片

## A.1 一句话定位

> **当前 UI 是核磁共振机：精密、强大、专业，但谁都不想躺进去。**
> **V5.0 要把它改成一支被精心削过的铅笔。**

**应用场景**：每次开发会上写在白板顶端。任何争论扯到"这个细节做不做"时，问一句——这是核磁共振机的逻辑，还是铅笔的逻辑？

---

## A.2 五条哲学锚句

任意 PR 描述、任意决策犹豫、任意"要不要加这个开关"的讨论，回到这五句话上判断：

1. **动人不可量化**——score、density、ratio 这类数字只判"管道是否漏水"，不判文学好坏。
2. **AI 帮不到顶尖文学，但可以避免它误伤顶尖文学**——所有诊断的默认动作是"提醒"，不是"命令"。
3. **Great design knows when to disappear**——写作时 UI 必须消失，AI 站在墙边而不是对面。
4. **数据不是给你看的，数据是替你做了一次判断、然后给你一句话**——15 个 metric 数字 → "第 7 章在等你"。
5. **藏得好的功能，用户感觉是"刚好需要时它就在"**——所有改造的本质是"把已有功能藏起来"，不是加新功能。

## B.1 第001章 59 分误判 · 整套改造的北极星

**原文**（[02_正文/第001章_草稿.md](02_正文/第001章_草稿.md)）：

> 雨停在凌晨两点。窗缝里还挂着水声，像有人把一串旧钥匙慢慢拖过墙面。
>
> 主角站在桌前,没有立刻碰那只信封。纸面被水汽泡皱,边角却干净得过分,像刚从某个不该存在的抽屉里取出来。
>
> ……地板上留下一小摊水,形状像一枚被压扁的眼睛。

**当前诊断**（[04_审核日志/第001章_质量诊断.json](04_审核日志/第001章_质量诊断.json)）：
- score: **59（需打磨）**
- conflict_signal_density_per_1k: **0**
- body_emotion_density_per_1k: **0**
- 系统建议："增加选择/代价/拒绝中的至少一种"

**问题**：这是全项目**最克制、最有质感**的一段中文。系统因为没有"冲突词"和"身体动作词"判它低分，**会逼着模型把它改坏**。

**对应 V5.0 任务**：
- **CORE T1.1** score_caveat 标签——这种段落要标"参考分"
- **CORE T2.1** ChapterMode=`atmosphere`——这章不该按 plot 阈值评估
- **CORE T2.3** writer_overrides——作家可以一键豁免"冲突弱"诊断
- **CORE T3.2** LiteraryView.memorable_moments——必须能识别"被压扁的眼睛"是可记忆瞬间
- **CORE T3.4** style_court——把质量诊断的低分降级，把文学批评的高分顶上来

**验收的真正口径**：V5.0 完成后再跑一遍这章，**confirmed_issues 应只剩"任务卡核心事件未在正文出现"这类硬约束，冲突/动作类全部进入 contested_issues**。

## B.2 郁时谌的双层人格 vs 诊断器 · 文学设定与工程指标的根本冲突

**原文**（[00_世界观/角色档案/郁时谌.md](00_世界观/角色档案/郁时谌.md)）：

> **底层型**：INTJ——系统性思考者
> **表层壳**：二十年银行体制把他磨成了表面 ISFJ：守规矩、不出头、把判断藏在沉默里。
> 读者在前期看到的是"银行壳"——温吞、程序化、不动声色；
> 口头禅"这个事情，流程上……"的出现频率递减，是这条弧线的可量化标尺。

**问题**：人物前 30 章的核心魅力是**不动声色**。但当前诊断会在每一章都告诉模型"冲突信号不足、agency 不够、身体情绪密度=0、建议增加选择/拒绝/反击"。

**两条路径必有一败**：
- 跟诊断走 → 人物从第 1 章就外显冲突 → 角色档案的"双层模型"垮掉
- 跟档案走 → 每章诊断红色 → 作家被噪音轰炸

**对应 V5.0 任务**：
- **CORE T2.1** ChapterMode + **CORE T2.2** StyleProfile——前 30 章可设 `interior` 模式
- **CORE T3.4** style_court——文学批评识别"克制的人物"，覆盖工程诊断
- **CORE T4.2** 作家裁决面板——作家直接给"冲突弱"建议打"我保护这处"

**验收口径**：开发完成后，给一个虚构章节（郁时谌全程沉默、用流程语言遮蔽真实判断），跑诊断 → top_3_must_fix 中**不应**出现"补冲突信号""补 agency""补 body_emotion"。

# 附录 C · 技术坑预警与对标产品索引

## C.1 Streamlit 的四个原生限制（必须在动手前理解）

| 限制 | 影响任务 | Workaround |
|---|---|---|
| `st.rerun()` 会丢失滚动位置和展开状态 | UX T1.4 后台任务、T2.1 单栏写作 | 用 `st.empty()` 局部刷新 + `st.session_state` 维护视图状态 |
| 按钮无 hover 效果、无 CSS 过渡 | UX T4.2 微动效 | 用 `st.markdown(unsafe_allow_html=True)` 注入自定义 HTML 按钮，或写 component |
| 不支持自定义快捷键 | UX T2.3 ⌘P / ⌘. / ⌘K | 用 `streamlit-shortcuts` 包，或用 `components.v1.html` 写一段 keydown 监听 + `postMessage` |
| `st.session_state` 跨线程访问会崩 | UX T1.4 background_jobs | 用 `threading.Event` + 共享变量 dict，主线程轮询；不要在子线程内 `st.session_state[k] = v` |

## C.2 必须做自定义 component 的三个地方

Streamlit 标准组件无法实现，必须用 `streamlit.components.v1` 写或引入第三方：

1. **UX T1.2 inline diff** — 段级 diff + 单段接受/拒绝 → 推荐 `streamlit-ace` 或自写 React 组件
2. **UX T2.2 margin notes** — 稿纸正文 + 右侧批注（hover 联动）→ 必须自写组件，约 1.5 天
3. **UX T2.3 keyboard shortcuts** — 全局快捷键监听 + 命令面板 → 必须自写组件，约 1 天

**这三块占 UX 改造总风险的 70%**。提前为它们留 buffer：T2.2 实际工时约 5 天（不是文档写的 4 天）。

## C.4 三个 StyleProfile 的人格摘要（CORE T2.2 必须按这个写）

**禁止做成三个长得一样的 profile**。每个 profile 必须有独立的灵魂：

### 金庸路线 `jin_yong`
- **valued_traits**: 短句、白描、动作叙事、对白潜台词
- **devalued_traits**: 长句、意识流、意象密集、心理直陈
- **样本来源**: 当前 `style_seed_library.md` 8 段
- **代表桥段**: 胡逸之关于陈圆圆的那段独白（极克制、几乎不动）
- **典型 cliche_override**: "深吸一口气" 在 high_tension 场景容忍

### 王小波路线 `wang_xiaobo`
- **valued_traits**: 机锋、跳跃叙事、戏谑、反讽、突然转弯
- **devalued_traits**: 工整对仗、抒情铺陈、宏大叙事
- **样本来源**: 《黄金时代》开篇 3 段、《红拂夜奔》随笔体
- **代表特征**: 思维比情节走得快，逻辑链条故意打折叠
- **典型 cliche_override**: "陷入沉默" 完全允许（王常用，且为反讽工具）

### 余华路线 `yu_hua`
- **valued_traits**: 重复、节制、残酷的诗意、白描血色
- **devalued_traits**: 心理直陈、情绪解释、修辞堆砌
- **样本来源**: 《活着》"福贵讲述"段落、《许三观卖血记》对白
- **代表特征**: 用最平的语言写最痛的事，让读者自己疼
- **典型 cliche_override**: "说不出话来" 允许（在重大变故场景是节制而非偷懒）

**验收口径**：选不同 profile 后，向同一个章纲生成正文，**三份输出读起来必须像三个作家写的**。如果像同一个人，profile 设计就失败了。

```python
# webui_infra/pages/continue_writing.py
def _read_last_writing_state(project_dir: Path) -> tuple[int, str, str] | None:
    """扫描 02_正文/*.md 按 mtime 排序取最新章节，返回末段引文。"""
    files = sorted(
        (project_dir / "02_正文").glob("第*章_*.md"),
        key=lambda p: p.stat().st_mtime,
        reverse=True,
    )
    for path in files:
        text = path.read_text(encoding="utf-8").strip()
        if not text:
            continue
        chapter_num = _extract_chapter_num(path.name)
        last_para = _last_non_empty_paragraph(text)
        mtime = datetime.fromtimestamp(path.stat().st_mtime).strftime("%H:%M")
        return chapter_num, last_para[-200:], mtime
    return None
```
