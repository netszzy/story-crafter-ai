# V5.0-alpha2 附录对照复检

> 复检日期：2026-05-06
> 对照文档：`docs/V5.0_DESIGN_SOUL_APPENDIX.md`
> 复检范围：ChapterMode、StyleProfile、writer overrides、EditorMemo reservations、正文生成 prompt 和写作页/设置页入口。

## 1. 总结论

V5.0-alpha2 已把 alpha1 的“诊断承认局限”推进成可持续的数据链路：章节模式、风格档案和作家豁免不再只是临时字段，而是进入任务卡、正文 prompt、质量诊断、编辑备忘录和改稿 prompt。

对照附录，本轮重点改善 B.1 和 B.2：`interior/atmosphere` 章节的 conflict 下限为 0；`open/echo` 结尾允许余味；王小波 profile 允许“陷入沉默”，但不会放过“涌上心头”；作家拒绝过的诊断会变成 `accepted_by_writer`，不再进入扣分和改稿建议。

仍未完成：真正的作家裁决面板还只是写作页质量详情里的轻量入口；LiteraryView 与 style_court 仍在 beta/rc；段级 inline diff 组件、页边批注和卷轴健康图仍未落地。

## 3. 关键变更

- `novel_schemas.py`：`ChapterTaskCard` 增加 `chapter_mode`、`ending_style`、`pacing`、`style_profile`；`EditorMemo` 增加 `reservations`、`style_profile`、`chapter_mode`。
- `style_profiles.py`：新增三套风格档案和 profile 样本入口。
- `prompts/style_profiles/`：新增三份原创教学样本，每份 5 段。
- `prompt_assembly.py`：正文 system prompt 注入章节模式规则、风格档案和 profile 样本。
- `quality_diagnostics.py`：按 ChapterMode 查阈值；运行时合并 profile cliche overrides；新增 writer override 读写与应用。
- `editor_memo.py`：过滤 `accepted_by_writer` findings，输出 reservations，并在改稿 prompt 中声明“禁止执行”。
- WebUI：设置页可保存项目风格档案；任务卡页可编辑 V5.0 章节口径；写作页质量详情可记录“我保护这条”。

## 4. 下一版本前硬门槛

1. beta1 做单栏稿纸和页边批注前，必须避免让诊断面板重新变成驾驶舱。
2. writer overrides 需要从轻量按钮升级为正式“作家裁决面板”，支持采纳、保护、反驳三态。
3. LiteraryView/style_court 落地前，不得把 alpha2 的启发式保护宣称为“文学批评已完成”。
4. 卷轴健康图必须替换三色热力图，不再用红黄绿审判式 UI。
5. 每个后续阶段都要继续更新附录对照复检文档。
