## 一、P0-B 风格样本注入（最先做）

### 1.1 数据流

```
   ┌──────────────────────────────────────┐
   │ 已定稿章节 (02_正文/定稿/*.md)        │
   │ + 戏剧诊断 cinematic_score           │
   └────────────┬─────────────────────────┘
                │
                ▼
   抽样器（按 cinematic_score 降序，取 top-3 段落）
                │
                ▼  （冷启动 fallback）
   inject_prose_samples()  ◀── 内置种子库（prompts/style_seed_library.md）
                │
                ▼
   prompt_assembly.assemble_chapter_prompt()
                │
                ▼
   prompts/正文生成.md 的 {{ style_samples }} 占位符
```

### 1.2 抽样规则

按章节阶段分三档：

| 章节阶段 | 样本来源 | 兜底策略 |
|---|---|---|
| 第 1-2 章（无定稿、无戏剧诊断） | 内置种子库（`prompts/style_seed_library.md`）取 3 段 | 用户可在 `00_世界观/文风档案.md` 添加"示范段落"区，优先级高于种子库 |
| 第 3-5 章（有定稿但样本少） | 已定稿章节内随机抽 3 段（80-200 字、含对白或动作） | 不足时用种子库补齐 |
| 第 6 章起 | 按 cinematic_score 降序取 top-3，**同段不重复** | 用户可在 WebUI 标记"锁定/排除"特定章节 |

抽样过滤条件（硬性）：
- 段落长度 80-200 字
- 必须含对白（`"…"`）或动作动词（避免抽到全景描写）
- 排除已被 LLM 标记为 `is_mock=True` 的章节段落

### 1.3 内置种子库（新建 `prompts/style_seed_library.md`）

设计原则：
- **自创教学样本**，规避版权风险
- 每段 80-150 字，标注使用的技巧
- 覆盖 8 种核心写作技巧

```markdown

> 用途：为新项目第 1-2 章提供风格锚点，避免 LLM 在零样本下生成翻译腔。
> 用户可用 `00_世界观/文风档案.md` 的"示范段落"区覆盖本库。

## 样本 1（技巧：身体动作代替情绪词）
他把杯子推得太用力，水洒在桌布上洇开一片。她没看他，只用指尖把那滩水按平，又把杯子摆回原位。"你说吧。"她说。两个人都没动。茶馆外面有车按喇叭，按了两下，停了。

## 样本 2（技巧：对白潜台词）
"你今天又去了那里。"
"嗯。"
"上次说好的。"
"我知道。"
"你要是再去——"
"我说我知道了。"
他放下筷子，碗里的饭没动几口。她看了他一眼，把汤推过去："先喝点。"

## 样本 3（技巧：感官细节叠加，慢节奏）
雨从下午三点开始下，到傍晚还没停。屋檐的水滴打在铁皮垃圾桶上，叮，叮，每一声之间隔着两秒。屋里没开灯，他坐在窗边，看雨把对面那栋楼的轮廓泡得发软。烟头烧到手指他才察觉，按熄在窗台上，留下一个黑点。

## 样本 4（技巧：短句节奏，紧张场景）
脚步声。近了。她屏住呼吸。门把手转了半圈，停。又转。她看见自己的影子在墙上抖。门被推开一条缝。光进来。然后是一只手。

## 样本 5（技巧：长句节奏，舒缓场景）
那是一个所有事情都还没发生的下午，阳光把院子里那棵老槐树的影子拉得很长，长到爬上厨房的窗台、又爬过母亲正在切菜的手背，而她什么都不知道，只是把切好的葱花拨进油锅，听着噼啪一声响起。

## 样本 6（技巧：环境描写带情绪，不直说）
巷口的雾把路灯啃成一团黄毛。他每走一步，影子就被吞掉一截，到了第三盏灯下面，连脚都看不见了。前面那个人也在走，距离不变，二十米。他停，那个人也停。他走，那个人也走。

## 样本 7（技巧：物品细节带象征）
钥匙放在玄关那个木碗里，已经八天了。她每天经过都看一眼，没动它。第九天早上她走过去，拿起来，又放回去。下午回家时，碗是空的。

## 样本 8（技巧：意象化心理，用具体物代替抽象词）
他想起十年前那个夏天，想起那条河，想起河水其实是浑的、岸边有死鱼的味道。可是在记忆里，那条河永远是清的。他知道自己在骗自己，但他不打算停。
```

### 2.2 JSON Schema（添加到 `novel_schemas.py`）

```python
class SceneTension(BaseModel):
    """单场戏的压力分析。"""
    scene_index: int                            # 第几幕（1-based）
    scene_summary: str = ""                     # 一句话概括（≤30 字）
    must_do: str = ""                           # 主角"必须做的选择"
    cost_if_fail: str = ""                      # 不做的代价
    pressure_level: int = Field(ge=0, le=10)    # 压力打分
    pressure_clarity: int = Field(ge=0, le=10)  # 读者能否感知压力


class CharacterArcSignal(BaseModel):
    """本章主要人物的弧光信号。"""
    name: str
    flaw_or_desire: str = ""                    # 角色档案里的致命弱点/核心欲望
    engaged: bool = False                       # 本章是否触碰了
    evidence_quote: str = ""                    # 触碰的原文摘录（≤80 字）
    arc_movement: Literal["前进", "停滞", "倒退", "未涉及"] = "未涉及"


class CinematicSample(BaseModel):
    """随机抽样段落的画面感评估。"""
    paragraph_index: int                        # 段落序号
    excerpt: str                                # 原文（≤120 字）
    visual_score: int = Field(ge=0, le=10)
    auditory_score: int = Field(ge=0, le=10)
    body_action_score: int = Field(ge=0, le=10) # 身体动作密度
    abstract_word_ratio: int = Field(ge=0, le=10) # 抽象情绪词占比（越低越好）
    rewrite_hint: str = ""                      # 一句话改写建议


class DramaticDiagnostics(BaseModel):
    """章节戏剧结构诊断报告（V3.0 新增）。"""
    chapter_number: int
    title: str = ""
    model_used: str = ""                        # 评估时用的模型 ID，便于版本追溯
    provider_used: str = ""                     # critic provider 名（deepseek/openrouter/...）
    generated_at: str = Field(default_factory=now_iso)

    # 三大维度评分（0-100）
    pressure_curve_score: int = Field(ge=0, le=100)
    character_arc_score: int = Field(ge=0, le=100)
    cinematic_score: int = Field(ge=0, le=100)
    overall_drama_score: int = Field(ge=0, le=100) # 加权平均

    # 详情
    scenes: list[SceneTension] = Field(default_factory=list)
    characters: list[CharacterArcSignal] = Field(default_factory=list)
    cinematic_samples: list[CinematicSample] = Field(default_factory=list)

    # 改稿优先级建议（喂给 reviser）
    top_revision_targets: list[str] = Field(default_factory=list)

    # Mock 标记
    is_mock: bool = False
```

文件路径：
- JSON：`04_审核日志/第NNN章_戏剧诊断.json`
- Markdown 摘要：`04_审核日志/第NNN章_戏剧诊断.md`
- 历史备份：`04_审核日志/versions/`（沿用既有备份机制）

### 2.3 Prompt 模板（新建 `prompts/戏剧诊断.md`）

```markdown
# 系统提示：戏剧结构诊断

你是一位资深小说编辑，专长于"故事结构、人物弧光、画面塑造"评估。
本次任务：对一章正文做**戏剧结构诊断**，输出严格的 JSON。
**禁止文学评论腔，禁止"很好/不错"这类泛泛评价；只给可量化结论 + 可执行改写建议。**

## 项目核心约束（用于对齐评判标准）
{{ axis_context }}

## 评估维度

### 1. 场景压力曲线（pressure_curve）
把本章按"场景"切分（通常 2-5 个），每个场景判断：
- 主角"必须做"的选择是什么？（must_do）
- 不做的代价是什么？（cost_if_fail）
- 压力是否上升？读者能否感知（pressure_clarity）？
- **判定原则**：如果 must_do 和 cost_if_fail 都说不清，pressure_level 必 ≤3。

### 2. 人物弧光（character_arc）
对照下方"本章主要人物档案"，判断每个人物本章是否触碰了"致命弱点 / 核心欲望"：
- engaged=true 时必须给 evidence_quote（原文摘录，≤80 字）
- engaged=false 时该人物本章是"工具人状态"

### 3. 画面可视性（cinematic）
随机抽 5 段（首/中/中/中/尾各 1），对每段评分：
- visual / auditory / body_action：越高代表越能被电影化
- abstract_word_ratio：含"复杂的情绪""莫名""说不出"等抽象词的密度（越低越好）
- 给一句话改写建议（rewrite_hint），示范如何把抽象写法换成具体可视写法

## 综合评分（必须按公式计算，禁止凭感觉打分）
- pressure_curve_score = 各场景 (pressure_level + pressure_clarity)/2 的均值 × 5
- character_arc_score = engaged=true 的人物比例 × 100，且对每个 arc_movement 为"倒退/停滞" 的扣 10 分
- cinematic_score = 5 个样本的 (visual + auditory + body_action) 均值 × 10/3，再扣 abstract_word_ratio 均值 × 5
- overall_drama_score = 0.4 × pressure + 0.35 × arc + 0.25 × cinematic

## top_revision_targets
最多 5 条，每条格式："[位置]：[问题]，[改法]"
示例："场景3：林望面对长老时压力不可见，补一句'你拒绝就明天送回去'让代价显形"

## 输出要求
- 只输出 JSON，符合下方 schema，禁止 Markdown 代码块包裹
- 所有评分必须是整数
- evidence_quote / excerpt 必须是原文摘录，不要复述

## JSON Schema
{{ json_schema }}
```

### 2.4 代码骨架（新建 `dramatic_arc_diagnostics.py`）

```python
"""
V3.0 戏剧结构诊断器
- 用 NOVEL_CRITIC_PROVIDER 调 LLM 评估"故事张力 / 人物弧光 / 画面感"
- 与 quality_diagnostics.py 共存，互补不替代
"""
from pathlib import Path
from llm_router import LLMRouter
from novel_schemas import (
    DramaticDiagnostics, SceneTension,
    CharacterArcSignal, CinematicSample,
)
from prompt_assembly import build_axis_context

PROMPT_PATH = Path(__file__).parent / "prompts" / "戏剧诊断.md"


def diagnose_chapter_drama(
    project_dir: Path,
    chapter_num: int,
    chapter_text: str,
    task_card_json: str,
    character_briefs: str,
    llm: LLMRouter,
) -> DramaticDiagnostics:
    """主入口：返回 DramaticDiagnostics 模型。"""
    system_prompt = _build_system_prompt(project_dir)
    user_msg = _build_user_msg(chapter_text, task_card_json, character_briefs)

    if llm.mode == "mock" or _should_mock(llm):
        return _mock_diagnostics(chapter_num, chapter_text)

    raw = llm.critic_text(  # 复用现有 critic 路由
        system_prompt=system_prompt,
        user_prompt=user_msg,
        workflow="dramatic-diagnose",
        role="dramatic-critic",
        max_tokens=4000,
    )
    return _parse_response(raw, chapter_num, llm)


def write_diagnostics(project_dir: Path, diag: DramaticDiagnostics) -> tuple[Path, Path]:
    """写入 JSON + Markdown 摘要，返回两个路径。
       Markdown 中包含雷达图原始数据 + top_revision_targets 列表。"""


def diagnostics_to_revision_brief(diag: DramaticDiagnostics) -> str:
    """把诊断转成给 reviser 的'改稿任务清单'，喂入 revise_text。
       优先级：戏剧诊断改法 > 质量诊断改法 > AI味改法。
       
       输出示例：
         ## 戏剧诊断改稿任务（按优先级）
         1. 场景3 压力不可见 → 在'他走到她面前'后补一句'你拒绝就明天送回去'
         2. 画面5 抽象情绪词过多 → '莫名的烦躁' 改为 '他把杯子推得太用力，水洒了出来'
    """


def _parse_response(raw: str, chapter_num: int, llm: LLMRouter) -> DramaticDiagnostics:
    """解析 LLM 返回的 JSON：
       - 容错：去除可能的 ```json``` 包裹
       - 失败时重试 1 次
       - 二次失败降级到 _mock_diagnostics() 并记录原文到 logs/dramatic_diagnose_failures/"""


def _mock_diagnostics(chapter_num: int, text: str) -> DramaticDiagnostics:
    """Mock：用句长/段落数等本地启发式给保守评分，便于离线验收。
       is_mock=True，避免被风格样本抽样器选中。"""


def _should_mock(llm: LLMRouter) -> bool:
    """根据 critic provider 的 API Key 状态判断是否走 mock。"""
```

### 2.6 Mock fallback 与成本

| 情况 | 行为 |

## 六、风险与回滚

| 风险 | 概率 | 缓解 |
|---|---|---|
| 戏剧诊断 LLM 输出 JSON 解析失败 | 中 | 重试 1 次 + 失败降级 mock + 原文存盘；单元测试覆盖 4 种异常 case |
