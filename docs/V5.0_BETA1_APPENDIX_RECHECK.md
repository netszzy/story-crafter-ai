# V5.0-beta1 附录对照复检

> 复检日期：2026-05-06
> 对照文档：`docs/V5.0_DESIGN_SOUL_APPENDIX.md`
> 复检范围：单栏稿纸、页边批注、快捷键唤出式面板、卷轴式全书健康图。

## 1. 总结论

V5.0-beta1 把 alpha2 的“文学保护数据链路”推进到写作体验层：写作页默认不再把章节列表、操作按钮和诊断雷达同时压到作家面前，而是回到单栏稿纸；诊断变成页边批注；命令、AI 和诊断以按钮或快捷键唤出；全书健康从 32x32 红黄绿方块改为卷轴式色带。

对照附录，本轮重点回应 B.3 和 B.4：`采纳此改法` 的 inline 预览仍保留，新增页边批注的「不用」会直接写入作家豁免；旧健康热力图被 `webui_infra/components/scroll_health.py` 替换为奶油到深褐连续映射，并提示「第 N 章在等你」。

仍未完成：页边批注还不是完整自定义 React 组件，hover 与正文段落的视觉联动是轻量版；快捷键通过 query param 桥接，受 Streamlit iframe 能力限制；真正的作家裁决面板、LiteraryView/style_court 和逐段接受/拒绝 diff 仍在后续 beta2/rc1。

## 2. 附录逐项对照

| 附录要点 | beta1 状态 | 结论 |
|---|---|---|
| A.2 Great design knows when to disappear | 写作页默认稿纸优先，命令/AI/诊断收起 | 达标推进 |
| A.2 数据替你做判断 | 卷轴健康图直接提示「第 N 章在等你」，不展示一排总分方块 | 达标推进 |
| B.3 “采纳此改法”六步真相 | inline 预览保留；页边批注「不用」进入 writer override，减少辅助稿路径负担 | 部分达标 |
| B.4 32x32 马赛克 | 红黄绿方块替换为 8px 横向卷轴色带，色彩为奶油到深褐 | 达标推进 |
| C.1/C.2 Streamlit 限制 | 快捷键与批注采用轻量 component/workaround，未宣称完整原生体验 | 风险可控 |

## 3. 关键变更

- `webui_infra/pages/writing.py`：写作页主入口改为顶部细工具栏 + 单栏稿纸；章节命令、AI 助手、诊断抽屉和专注模式由状态控制。
- `webui_infra/components/margin_notes.py`：把 `polish_targets` 与 `findings` 聚合为最多 5 条段落批注，跳过 `accepted_by_writer`。
- `webui_infra/components/keyboard.py`：接入 `Ctrl/Cmd+P`、`Ctrl/Cmd+.`、`Ctrl/Cmd+K`、`Ctrl/Cmd+Enter`、`Ctrl/Cmd+Shift+F`。
- `webui_infra/components/scroll_health.py`：收集戏剧/质量诊断，渲染卷轴健康图和浮卡数据。
- `webui.py`：旧 `_render_chapter_health_heatmap()` 改为调用卷轴健康组件，侧栏版本标识更新为 `V5.0-beta1`。
- `webui_infra/state.py`：新增写作表面状态默认值。

## 4. 下一版本前硬门槛

1. beta2 落 LiteraryView/style_court 时，不能把工程诊断重新抬回文学裁判位置。
2. rc1 作家裁决面板需要把「采纳 / 保护 / 反驳」三态正式化，页边批注的「不用」只是轻量入口。
3. inline diff 仍需从整稿采用推进到逐段采用/丢弃。
4. 如果后续引入自定义 React component，要优先补 hover 段落联动和滚动位置保持。
