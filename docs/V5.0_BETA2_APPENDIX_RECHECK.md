# V5.0-beta2 附录对照复检

> 日期：2026-05-06
> 范围：文学批评层、风格法庭、EditorMemo 接入、CLI/WebUI 展示
> 依据：`docs/V5.0_UPGRADE_PLAN.md` 第 7 节，`docs/V5.0_DESIGN_SOUL_APPENDIX.md` A/B 章

## 复检结论

V5.0-beta2 已完成“文学批评层与风格法庭”的最小闭环：系统不再只让质量分和工程 finding 直接推动改稿，而是先生成 LiteraryView，再由 style_court 判断哪些建议是硬约束、哪些建议可能误伤文学性。

这次升级符合附录 A.2：

- 动人不可量化：LiteraryView 不输出 score，只输出可记忆瞬间、未说之语、残响和文学风险。
- AI 避免误伤文学：风格法庭会把冲突、主动性、身体情绪等可能破坏氛围/内省的建议降入 contested。
- 作家裁决权：contested 自动进入 EditorMemo reservations，改稿 prompt 明确禁止执行。

## B.1 第001章 59 分误判

反证要求：

- “被压扁的眼睛”必须能被识别为可记忆瞬间。
- `chapter_mode=atmosphere` 且 `cannot_be_quantified=true` 时，冲突/动作/身体情绪类建议应进入 contested。
- 任务卡核心事件、forbidden 等硬约束仍应保留在 confirmed。

已落实：

- `literary_critic.py` 的 mock 验收能识别“被压扁的眼睛”，并标注 `[Mock] 未调用真实文学批评模型`。
- `style_court.py` 在 atmosphere/interior/bridge 且 LiteraryView 不可量化时，将“冲突信号偏弱”“情绪身体化偏弱”等软工程建议放入 `contested_issues`。
- “任务卡对齐不足”“触碰任务卡禁止事项”仍进入 `confirmed_issues`。
- `editor_memo.py` 会把 contested 写入 reservations，避免再次进入 `top_3_must_fix` 或 revision prompt。

专项测试：

- `tests.test_literary_critic.LiteraryCriticTests.test_mock_finds_appendix_b1_moment_and_marks_mock`
- `tests.test_literary_critic.StyleCourtTests.test_style_court_contests_soft_findings_but_confirms_hard_constraints`
- `tests.test_literary_critic.StyleCourtTests.test_editor_memo_fallback_keeps_contested_out_of_must_fix`

## B.2 郁时谌双层人格

反证要求：

- 克制、流程语言、沉默和不动声色不能被诊断器持续轰炸。
- top_3_must_fix 不应出现“补冲突信号”“补 agency”“补 body_emotion”。

已落实：

- LiteraryView 会记录 `self_deception_signals` 和 `unsaid_tension`，用于表达流程、规矩、沉默背后的遮蔽。
- style_court 对内省/氛围章节开启不可量化保护后，会把主动性、冲突和身体情绪类 finding 归入 contested。
- EditorMemo fallback 与 LLM prompt 都明确：contested 不得换说法进入必改项。

剩余风险：

- 真实 LLM 生成的 LiteraryView 仍需要人工回归第 001 章和郁时谌样章；当前自动测试覆盖的是 mock 与规则裁决。
- WebUI 已能展示文学批评和风格法庭，但“逐条作家裁决面板”仍属于 V5.0-rc1。
