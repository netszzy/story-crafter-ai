请根据世界设定和总纲，批量生成 {{ count }} 个角色档案草案。

重要：批量角色必须围绕项目轴设计，尤其是“故事规格”里的主角、对手、情感牵引者、信息持有者、类型卖点和成功标准。
数量硬约束：必须恰好输出 {{ count }} 个角色，不要多、不要少；每个角色都必须能被系统拆分成独立档案。

## 输出格式
每个角色必须用以下一级标题开始，便于系统自动拆分：

```markdown
# 角色档案：角色名
```

## 输出要求
- 使用 Markdown。
- 必须恰好生成 {{ count }} 个独立角色档案。
- 每个角色开头必须包含“项目规格对齐”小节，说明该角色服务哪条故事规格。
- 每个角色都必须包含：基础信息、外貌不可变特征、核心驱动、恐惧、道德边界、说话方式、绝不会说的话、标志性动作、秘密、关系钩子、当前状态。
- 角色之间必须有关系张力，至少包含：主角/推动者、阻碍者、信息持有者、情感牵引者、灰度盟友。
- 不要复用已有角色名。
- 不要写成名单简介，每个角色都要达到可直接进入角色档案的完整度。
- 不要直接改写正式档案，只输出草案。
- 不要生成无法进入主线结构或人物关系网的孤立角色。
